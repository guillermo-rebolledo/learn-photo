# Ship analytics behind a pluggable, non-blocking provider

Anonymous curriculum analytics (`lib/analytics-model.ts`, `lib/analytics.ts`) is implemented as a validated event contract dispatched through a swappable `AnalyticsProvider`, not a specific vendor SDK. Every event is checked against an allowlist of canonical identifiers before it can reach a provider, so no name, typed content, personal imagery, advertising identifier, or cross-site identifier can be sent even by mistake. Dispatch is synchronous, wrapped in `try`/`catch`, and no-ops outside the browser, so a missing, blocked, or failing provider never affects the Learning Loop, Progress, or rendering (ADR 0009). The shipped default provider delivers events with `navigator.sendBeacon` to a same-origin path (`/analytics/events` by default, overridable at build time via `NEXT_PUBLIC_ANALYTICS_ENDPOINT`), which keeps analytics compatible with the static export (ADR 0004, ADR 0011) without introducing a runtime dependency; no collector exists yet at that path, so events are inert in production until one is deployed as a follow-up, which is acceptable because analytics is explicitly non-blocking and omittable (MVP-SPEC).
