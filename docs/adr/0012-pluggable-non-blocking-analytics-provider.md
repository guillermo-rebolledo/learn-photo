# Ship analytics behind a pluggable, non-blocking provider

Anonymous curriculum analytics (`lib/analytics-model.ts`, `lib/analytics.ts`) is implemented as a validated event contract dispatched through a swappable `AnalyticsProvider`, not a specific vendor SDK. Every event is validated against the domain's registries of canonical Lesson, Challenge, Curated Scene, and Success Criterion identifiers — derived from the curriculum model itself, not a format check — before it can reach a provider, so a value that is not an agreed curriculum outcome (a name, typed content, or any non-curriculum string) is rejected even by mistake. Validation runs at the exported dispatch boundary and returns a fresh normalized copy, so an event mutated or `any`-cast after construction cannot slip through. Dispatch is synchronous, wrapped in `try`/`catch`, and no-ops outside the browser, so a missing, blocked, or failing provider never affects the Learning Loop, Progress, or rendering (ADR 0009). The shipped default provider delivers events with `navigator.sendBeacon`, which sends credentials, so it refuses any endpoint that is not same-origin: the endpoint defaults to the `/analytics/events` path and is overridable at build time via `NEXT_PUBLIC_ANALYTICS_ENDPOINT`, but a cross-origin override is dropped rather than allowed to carry the Learner's cookies off-origin. Same-origin delivery keeps analytics compatible with the static export (ADR 0004, ADR 0011) without introducing a runtime dependency; no collector exists yet at that path, so events are inert in production until one is deployed as a follow-up, which is acceptable because analytics is explicitly non-blocking and omittable (MVP-SPEC).
